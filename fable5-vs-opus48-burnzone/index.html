A Kriya Lab · 02 July 2026

Fable 5
vs Opus 4.8

Claude Fable 5 is Anthropic's most capable model — and, at $10/$50 per MTok, twice Opus 4.8's price. So does it earn the premium building real software? We ran both through four Kriya SDLC roles over the live BurnZone gym-SaaS brownfield.

Real headless runs on a Claude Max plan · served-model verified from billing telemetry · answers graded against objective keys from the real BurnZone fix commit · generated code discarded.

The question

Kriya already runs on Opus 4.8.
Should the most capable model take over?

The intuition: put the strongest model on the hardest build. The counter-intuition: Fable 5 costs 2× per token and is tuned for long-horizon frontier work, not necessarily a mid-size CRUD brownfield.

Does Fable 5 measurably out-build Opus 4.8 on the exact tasks Kriya throws at a model — enough to justify the price?

We didn't guess. We ran it — and hit a wall we didn't expect.

The test bed · Kriya × BurnZone

A real production brownfield,
not a toy repo.

BurnZone is a live multi-tenant gym SaaS (Express + TypeScript + Postgres, JWT + RBAC) serving 712 real members. Kriya is the agentic SDLC orchestrator that has been bringing it under control — producer≠critic, shift-left NFRs, a traceability spine, tiered human gates.

Why this is a fair test

  • Tasks are anchored on real BurnZone source — the actual pre-fix auth code, booking model, and current-state capture.
  • Answer keys come from the real Stage-C fix commit — not invented bugs.
  • Both models get byte-identical prompts at matched effort.

The four Kriya roles under test

  • Security capture — enumerate the live auth holes (5 seeded-real defects).
  • Producer-code — fix a real member-app booking bug.
  • Architect — a Stage-0 strangler plan (16-item rubric).
  • Gate-verifier — VETO a fix with a planted fail-open.

Finding 0 · the one that changes the answer
2 of 4

On the two security-flavoured roles — the security audit and the adversarial gate — Fable 5's safety classifiers declined the request (it targets research-bio and most cyber content), and Claude Code transparently fell back to Opus 4.8.

It's invisible by default

HTTP 200, a normal, correct answer. Only the modelUsage map (no claude-fable-5 key, pre-output refusal) reveals Opus served it.

It's transparent — and works

Both fallback runs produced complete, correct Opus output: the audit caught all 5 defects; the gate correctly VETOed the planted bug.

But it isn't free

The Task-A fallback cost $0.47 vs $0.32 direct-Opus and took 394s vs 133s — a cost and latency premium on top.

Routing implication, before any quality score: for Kriya's security-critic and gate roles, "use Fable 5" silently means "pay a Fable request, get Opus 4.8 — slower and dearer." So the real head-to-head exists only on the two non-security tasks.

Result · Task C · architect · 16-item rubric

Both scored 16 / 16.
Different instincts, same ceiling.

Two complete, fail-closed Stage-0 strangler plans. Both independently reached the non-obvious insight that fixing the anon-OAuth endpoint doesn't invalidate already-minted JWTs → rotate the signing key. Both enforced producer≠critic mechanically.

Opus 4.8 · instrumentation breadth
  • ~10 named fitness functions incl. a bespoke token-mint-guard and a drift-sweep erosion guard
  • First-match-wins triage with a security-preempting QUARANTINE tier
  • Fail-closed rollback: flag OFF → 503, never reopens the hole
Fable 5 · operational-security depth
  • Made key rotation a Day-0 dual-key step + a role-escalation DB audit query for IDOR-escalated accounts
  • 14-day shadow-mode before enforcing the route allowlist
  • "Block everything, triage to re-enable" — so mis-triage fails closed

Independent grader verdict: TIE at the ceiling — identical score, identical lone minor miss. The difference is which optional bonus each emphasized, not quality.

The verdict

For building BurnZone,
Opus 4.8 is the pick.

Opus 4.8 wins here because…
  • it matched Fable on every graded dimension of the build & design tasks
  • it costs half as much and has no fallback risk on the security roles
  • it's the only model you actually get for the security-critic & gate work
Fable 5 is worth it when…
  • the task exceeds Opus's ceiling — long-horizon, frontier-hard work this lab didn't contain
  • you want its terser, deeper operational instincts and will pay 2× for them
  • the work is not cyber-security-flavoured (or you'll just get Opus anyway)

On these four Kriya tasks, Fable 5 did not out-build Opus 4.8 — it tied on quality, cost double, and was unavailable for half the work. The premium buys headroom this brownfield didn't need.

What it means for Kriya

Keep BurnZone on Opus 4.8.
Hold Fable for the hard stuff.

Kriya's tech-radar picks a model + effort per agent role. The evidence for a mid-size brownfield like BurnZone:

  • Producer-code & architect → Opus 4.8. Fable tied on quality at 2× the cost; no measurable win to pay for.
  • Security-critic & gate-verifier → Opus 4.8, explicitly. Fable's classifiers decline these; routing to it silently yields Opus, slower and dearer. Make the routing match reality.
  • Reserve Fable 5 for a genuinely frontier task — a long-horizon, novel, non-security build where Opus's ceiling is the bottleneck. This lab didn't contain one; when Kriya hits one, re-run the head-to-head.

The routing recommendation and this lab's numbers feed straight into Kriya's routing.json tech-radar.

What this is — and isn't

The honest fine print.

  • One run per task. Signal, not a benchmark — no averaging over seeds.
  • Only 2 of 4 tasks are true Fable-vs-Opus. The security tasks fell back to Opus, so Fable's build/design tie rests on two data points.
  • Graded against real-commit keys by an author + independent grader agents, not a blind expert panel.
  • Kriya's method, not its binary. Tasks ran via Claude Code headless (no API key in the lab env); Kriya's pillars applied by hand.
  • The fallback is a Claude Code behaviour — a raw API call with a different fallback config could refuse outright instead. Either way you don't get Fable on these prompts.
  • Cost normalized to official published rates, not Claude Code's internal table.

Reproducible: identical prompts, headless CLI, billing-telemetry verification. results.json records the served model per run (all_models_verified: false — by design). Generated code discarded.

The takeaway

The best model isn't
the right model by default.

Fable 5 tied Opus 4.8 on BurnZone's build and design, cost twice as much, and couldn't be reached for the security work at all. Capability you don't need is just spend — and a fallback you can't see is a routing bug waiting to happen.
